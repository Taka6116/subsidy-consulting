# NTS 補助金サービス ゲートLP 設計書
## Cursor実装用デザイン＆コピー仕様

**作成日:** 2026年4月3日
**プロジェクト:** 株式会社日本提携支援（NTS）補助金サービス
**目的:** 経営課題診断ゲートLP — Cursorへの引き渡し用設計ドキュメント
**フレームワーク:** Next.js（App Router）+ Tailwind CSS + Framer Motion

## 1. トンマナ（トーン＆マナー）

### 1-1. デザインコンセプト

**キーワード:** 静かな信頼感 × 動きで引き込む

3つの参考サイトから抽出した設計思想：

| 参考サイト | 取り入れる要素 | 取り入れない要素 |
|-----------|--------------|----------------|
| Worker's Resort | ファーストビューでキーメッセージのみを大きく中央配置。余白の使い方。スクロールで情報が段階的に現れる構造 | 記事一覧型のレイアウト |
| CARTA ZERO | 全体のアニメーション品質。セクション遷移の滑らかさ。テキストの出現演出。スクロールに連動した動き | 暗い背景色。コーポレート感が強すぎるトーン |
| FiTO | 「関係をつくる」という温かみのあるコンセプト表現。循環・つながりを感じさせるビジュアル。明るく開放的な色調 | アパレル特有のビジュアル |

### 1-2. デザイン原則

```
1. First View = 1メッセージ + 1CTA のみ（Worker's Resort的潔さ）
2. スクロールで世界が開く（CARTA ZERO的な段階的開示）
3. 温かみ × 専門性（FiTO的な関係構築トーン）
4. 暗くしない — 背景は明るいオフホワイト系
5. 動きは「演出」ではなく「導線」として機能させる
```

**設計意図:**

- Neutral: 黄色みのあるオフホワイト（純白を避けることで柔らかい印象に）

### 1-5. アニメーション仕様

```
■ ファーストビュー
- キーメッセージ: フェードイン + 下から8pxスライド（duration: 800ms, ease: cubic-bezier(0.16, 1, 0.3, 1)）
- サブコピー: キーメッセージの300ms後にフェードイン
- CTAボタン: サブコピーの200ms後にフェードイン + スケール0.95→1.0
- 背景: 軽い粒子アニメーション or 抽象的な線のアニメーション（控えめに）

■ スクロール連動
- 各セクション: Intersection Observer で in-view 時にフェードイン（threshold: 0.15）
- 数字: カウントアップアニメーション（「最大1,500万円」など）
- カード群: stagger 100ms でカスケード表示

■ マイクロインタラクション
- CTAボタンホバー: 背景色トランジション 200ms + 軽い上方移動 2px
- 診断ステップ: プログレスバーのスムーズ遷移
- リンクホバー: 下線がスライドイン

■ 注意点
- prefers-reduced-motion を必ず尊重
- モバイルではスクロール連動アニメを軽量化
- パフォーマンス: transform と opacity のみアニメーション対象
```

```

┌─────────────────────────────────────────────────────┐
│  [NTSロゴ]                    [パートナー企業の方へ]  │
└─────────────────────────────────────────────────────┘

│        │  無料で診断する（1分）  │                    │
│        └──────────────────────┘                     │
│                                                     │
│        ▽ 3つの質問に答えるだけ                       │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**コピー:**

| 要素 | テキスト | 備考 |
|-----|---------|------|
| メインコピー（上段） | 人手不足、設備の老朽化、事業承継—— | var(--neutral-700), font-weight: 400, var(--text-h2) |
| メインコピー（下段） | その経営課題、国の支援制度で動かせるかもしれません。 | var(--primary-900), font-weight: 700, var(--text-hero) |

**デザイン詳細:**
- 高さ: 100vh（ビューポート全体）
- テキスト: 中央寄せ、max-width: 720px
- 背景: var(--neutral-100) にごく薄い抽象パターン or グラデーション
  - 案A: 右下から薄い円形グラデーション（var(--primary-100) → transparent）
  - 案B: ゆっくり動く細い線のSVGアニメーション（控えめに）
- CTAボタン: ページ内唯一の強いカラー要素。視線が自然に集まる設計
- 下部にスクロール誘導（chevron down アイコン、ゆっくりバウンスアニメーション）

**心理学的根拠（認知負荷の最小化 + ツァイガルニク効果）:**
- 情報を極限まで絞ることで「5秒テスト」をクリア
- 「動かせるかもしれません」の不確実性が好奇心を喚起し、診断への行動を促す

**設計思想:** ユーザーの「自分ごと化」を促す。FiTO的な温かみのあるトーンで、課題を列挙するのではなく問いかけとして提示する。

│   └─────────┘  └─────────┘  └─────────┘            │
│                                                     │
│     ┌─────────┐                                     │
│     │生産性を上│                                     │

```

**コピー:**

| 要素 | テキスト |
|-----|---------|
| セクション見出し | こんな課題を抱えていませんか？ |

**デザイン詳細:**
- 背景: var(--neutral-50)（白に近いオフホワイト）
- カード: var(--neutral-50) 背景、var(--neutral-200) ボーダー 1px、角丸 12px、影なし（フラット）
- カードレイアウト: デスクトップ4列、タブレット2列、モバイル1列
- アニメーション: スクロールin-viewでカードがstagger表示（100ms間隔）
- カードの上部にシンプルなアイコン（線画スタイル、24px）

**心理学的根拠（自己参照効果 + Pain-Agitate）:**
- 「こんな課題を〜」という問いかけ型は、読み手が自分の状況と照合する行動を自然に誘発
- 4つの課題は補助金の主要対象領域と完全に一致させている

```

### ファーストビュー ヘッドライン
